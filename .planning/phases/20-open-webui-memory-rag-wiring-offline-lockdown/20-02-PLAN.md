<objective>
Build the off-hardware code for the headline PRIV-05 gate: a runtime firewalled zero-outbound document-upload RAG smoke test that extends the Phase-19 proof seam (D-10). Implement a PURE `evalRagSmoke` core (unit-testable off-hardware via injected probes), a `liveRagSmoke` seam that drives the REAL OWUI REST RAG path (upload → embed via villa-embed → store in Qdrant → retrieve → assert citation, entirely through the local embed+Qdrant path — D-08) under host-level egress blocking paired with a negative-control external probe that MUST fail, and a `villa verify memory` subcommand to invoke it on-hardware. This plan delivers the testable slice of KB-01/02/03 + PRIV-05 and the harness that keeps the RAG path on the standard (citation-bearing) route — which is what enforces MEM-03's default-off auto-extraction (Native Function Calling stays OFF; only the manual `ENABLE_MEMORIES` store is wired).

Decision coverage in this plan: D-08 (upload→chunk→embed→retrieve runs entirely through villa-embed + Qdrant — the drive exercises exactly that local path), D-10 (runtime zero-outbound proof extending the Phase-19 seam, negative-control-first), D-11 (no new published host port — loopback drive + villa.network probe), D-07/MEM-03 (auto-extraction default-off enforced by keeping Native FC off).

Purpose: Install-time green is NOT sufficient for SC#4 — OWUI lazily fetches embed/reranker models from HuggingFace at runtime and ChromaDB posts PostHog telemetry. The runtime smoke test proves a real upload reaches no external host. Asserting zero-outbound by absence alone is a false-green; the negative control proves egress is actually blocked.
Output: A pure proof core + injected-probe unit tests (green off-hardware), the live seam, and a wired `villa verify memory` subcommand. The actual on-hardware drive runs in Plan 03.
