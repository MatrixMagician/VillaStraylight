<objective>
Parameterize the Open WebUI managed-service render view so the D-09 RAG/Qdrant/memory env block (per the PINNED 18-DECISIONS D-09 table, re-verified against OWUI source in 20-RESEARCH) is appended as a single ordered group after the existing entries (D-02) ONLY when `memory_enabled=true` (D-04), leaving a memory-off render byte-identical to the existing golden. This is the env-wiring half of INFRA-03 ("OWUI wired to Qdrant + local embeddings, config the single source of truth"), implemented entirely off-hardware as a single deliberate golden re-freeze (D-05).

Decision coverage in this plan: D-01 (multitenancy=True), D-02 (ordered appended group), D-03 (mandatory ENABLE_PERSISTENT_CONFIG=False), D-04 (conditional on memory_enabled / byte-identical off / config-sourced values), D-05 (single golden re-freeze + telemetry re-audit), D-06 (ENABLE_MEMORIES=True), D-09 (nomic prefixes), D-11 (no new published port).

Purpose: Without `ENABLE_PERSISTENT_CONFIG=False` plus the RAG/embedding/memory keys, OWUI silently ignores `config.toml` after first boot (the ConfigVar-shadows-env hazard — D-03). This plan makes config the authoritative source for OWUI's memory/RAG wiring and freezes that contract.
Output: A parameterized `buildOpenWebUIView`, a new memory-ON golden, a memory-aware telemetry test, and a render.go call-site change — all green under `make check`.
</objective>
