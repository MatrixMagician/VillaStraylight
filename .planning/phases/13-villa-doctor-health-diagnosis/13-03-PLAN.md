<objective>
Close the single diagnosed UAT gap (13-UAT.md Test 1): on a fully-healthy opt-in
ROCm install — offload residency PROVEN (log ROCm0 marker + sysfs GTT delta), all
/health 200, no config-vs-disk drift — `villa doctor` returns exit 2 (Overall WARN)
instead of exit 0, so a residency-proven ROCm install can NEVER reach exit 0.

Root cause (fully diagnosed in .planning/debug/doctor-healthy-rocm-exits-2.md): doctor.Aggregate
re-runs the STANDALONE pre-install ROCm host-prep gate (preflight.RunROCm) against a
POST-install running stack. RunROCmWithPolicy hardcodes firmware + HSA as typed-Unknown
(internal/preflight/checks_rocm.go:66-67) and RunROCm passes an empty requested image, so
checkROCmFirmware / checkROCmHSA / checkROCmImage take their "could-not-evaluate" WARN
branches unconditionally. The worst-wins fold rolls any single WARN up to Overall=WARN ->
exit 2. There is NO residency-supersession: a proven ROCm offload PASS does not down-rank
the un-evaluable host-prep WARNs.

Fix (Option A, primary; Option B, low-risk additive — both in scope):
- Option A: add a residency-supersession rule in the PURE doctor core (Aggregate). When a
  ROCm-family service's offload Verdict is inference.StatusPass (residency PROVEN), the
  typed-Unknown ROCm host-prep findings — matched by finding ID (ROCM-PRE-firmware /
  ROCM-PRE-hsa / ROCM-PRE-image) AND Status==WARN — must NOT raise the worst-wins rank.
  Keep them VISIBLE in Findings (non-rank-raising), ONLY under proven residency.
- Option B: thread the running ROCm image (inference.BackendFor(cfg.Backend).Image()) into
  the gate via the existing preflight.RunROCmForImage so a denied RUNNING image is actively
  caught (confident FAIL) rather than merely un-evaluated.

HARD INVARIANT (no-false-green, DOCTOR-02): a CONFIDENT bad signal must still BLOCK -> exit 1.
A Known deny-listed firmware (checks_rocm.go:140-146), Known-wrong HSA (189-194), or denied
image (219-225) is a confident FAIL and MUST remain rank-raising — even on the SAME ROCM-PRE-*
IDs the supersession otherwise down-ranks (the downgrade matches ID AND Status==WARN jointly,
never ID alone). Only the un-evaluable typed-Unknown host-prep WARNs that proven residency
already supersedes may be downgraded. Do NOT broaden the downgrade to non-ROCm-host-prep
findings, to drift, or to health.

Constraints honored: status.Report untouched (D-02). Read-only — no MkdirAll/WriteUnits (D-03).
No backend marker literal leaks into cmd/villa or internal/doctor (TestSeamGrepGate). Prefer
NO doctor golden schema change — supersession is rank logic, not a contract field; schema_version
stays 1.

Purpose: restore the DOCTOR-01 "exit 0 = healthy" contract on the recommended ROCm path
without weakening no-false-green (DOCTOR-02).
Output: a residency-superseding doctor core + image-threaded liveDoctorDeps, with supersession
and no-false-green-regression tests, all goldens still green.
</objective>
