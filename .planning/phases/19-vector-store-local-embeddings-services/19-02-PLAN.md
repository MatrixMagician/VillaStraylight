<objective>
Wire the two Plan-19-01 rendered services into the `villa install` lifecycle: (1) pre-stage
the pinned `nomic-embed-text-v1.5` Q8_0 GGUF into the `villa-models` volume at install via the
existing `internal/download` path so runtime is zero-download (D-07 / PRIV-04); (2) start
`villa-qdrant.service` + `villa-embed.service` after inference + Open WebUI, gated on
`memory_enabled`; (3) add a memory-stack readiness proof — an offline 768-length `/v1/embeddings`
smoke + a Qdrant writable round-trip — that refuses-with-remediation on failure (D-09, SC#3/SC#2,
honesty-by-construction). All three steps are no-ops when `memory_enabled=false` and under `--dry-run`.

The gate's authoritative `MemoryEnabled` is the PERSISTED config (`config.LoadVilla()`), threaded into
`runInstall` via a new `loadedMemoryEnabled` seam — NOT the `config.DefaultVillaConfig()` seed at
install.go:303 (which is always false). Binding the gate to the default seed would make the memory path
silently never fire for an opted-in user; this plan pins the authoritative source explicitly.

Purpose: deliver PRIV-04 (no runtime model download — pre-stage at install per D-07) and the
install-time half of INFRA-02 (the embeddings endpoint is proven to answer 768-dim before install
declares success).
Output: `cmd/villa/install_memory.go` (the nomic Shard + proof seam + live helpers), new `installDeps`
seams (incl. `loadedMemoryEnabled`), the memory steps threaded into `runInstall`, and table-driven
`install_test.go` cases (including an end-to-end memory-on `runInstall` test driven by a loaded config).
</objective>
