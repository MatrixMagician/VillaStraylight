<objective>
Wire the guided wizard into `villa install`: add the `--no-tui` flag, the `wizard` +
`stdoutIsTTY` seams on `installDeps`, the post-runChecks wizard branch in `runInstall`, the
`consentDecisions` threading through `gateInstall`/`resolveGap`/`offerNonBlockingGap`, the
`safeAutoFix` classifier (D-05), and the live huh 5-screen wizard runner in a new
`install_wizard.go`.

The composition is PINNED: `runInstall` already runs probe → pick → runChecks (steps 1-3)
BEFORE `gateInstall`. The wizard slots in between — it RECEIVES the already-computed
`detect.HostProfile`, `recommend.Recommendation` (+ alternatives), and `[]preflight.CheckResult`
as inputs (it computes none of them), COLLECTS a model override + per-item privileged consent,
and RETURNS to `runInstall`. The single `gateInstall` call then runs exactly once, consuming
the threaded consent. The wizard NEVER executes `runGapFix` itself. This guarantees probe/pick/
runChecks/gate each execute exactly once for BOTH paths (SC#1/SC#2 byte-identical config.toml;
privileged fixes run at most once; D-04/D-06 preserved).

Purpose: INSTALL-01 (presentation-only wizard composing the existing cores) + INSTALL-02
(graceful non-TTY / `--no-tui` fallback). Adds zero fit/preflight/backend decision logic.
Output: `cmd/villa/install_wizard.go` (new) + the branch/threading/classifier extensions in
`cmd/villa/install.go`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user (TTY) → wizard | Untrusted keyboard input selects a catalog model id + y/n consent |
| wizard consent → gateInstall (privileged) | A consented privileged command crosses to host execution — but ONLY via the single gateInstall→runGapFix call, never from the wizard |
| TUI text → seam | Backend/image names rendered in the review screen |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-01 | Elevation of Privilege | privileged fix via wizard consent (screen 3) | mitigate | D-04: privileged fixes ALWAYS per-item consent-gated; the wizard COLLECTS consent only and never calls runGapFix; the SINGLE gateInstall→resolveGap→runGapFix path is the only execution point, so a privileged fix runs AT MOST ONCE; `runGapFix` is fixed-arg, never a shell; safeAutoFix returns false for PRE-03/PRE-05; non-TTY/--json never reaches the wizard branch. |
| T-17-02 | Tampering | model name interpolation | mitigate | Model choice constrained to catalog ids from `recommend.Pick` options (no free-text into commands); override re-validated through Pick; ContainerArgs builds fixed-arg slices behind the seam. |
| T-17-04 | Tampering (contract erosion) | backend-literal leak via review text | mitigate | Render names via threaded `in.backend.Name()`/`.Image()` (or `BackendFor(...)`); `TestSeamGrepGate` walks `cmd/villa`. Config value `"vulkan"` is seam-clean. |
| T-17-05 | Denial of Service / unsafe mutation | accidental Enter on Install confirm | mitigate | D-07: Install confirm defaults focus to Cancel; mutation only on deliberate Install+Enter; Esc/Ctrl+C aborts non-zero with no mutation. |
| T-17-07 | Repudiation / double-execution | consent threaded into gateInstall | mitigate | Threaded `consentDecisions` is consumed by the SINGLE gateInstall call; a recorded decision suppresses the stdin re-prompt (huh already consumed stdin) but routes through the same fixed-arg runGapFix — no second prompt, no second execution; nil map = unchanged flag path. |
</threat_model>
