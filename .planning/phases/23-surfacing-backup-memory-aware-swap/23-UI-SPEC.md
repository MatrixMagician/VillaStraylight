# Phase 23 — UI Design Contract

> Visual and interaction contract for the Phase 23 dashboard surfacing work (CTRL-02 only — backup/restore and swap guards are CLI-side and carry no dashboard UI). Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope of this contract:** the existing embedded control dashboard (`internal/dashboard/assets/dashboard.html` / `dashboard.css` / `dashboard.js` — no-build vanilla JS, served via `go:embed`). Phase 23 adds: (a) `villa-qdrant` + `villa-embed` health rows (zero JS change — `renderHealth` auto-renders `report.services`), and (b) a new **Memory panel** reading `report.memory` (active embedding model + dimension, recall-index summary), shown only when `memory_enabled=true`. The design system below is the v1.0 Phase-5 contract as extended in Phases 10/15 — **detected from `dashboard.css`, reused verbatim, nothing new invented.**

## Design System

| Property | Value |
|----------|-------|
| Tool | none — hand-written `dashboard.css` with CSS custom properties (no preprocessor, no utility framework, no JS toolchain; established v1.0 Phase 5) |
| Preset | not applicable |
| Component library | none — in-house patterns: `.panel`, `.health-row`, `.metric-row`, `.badge`, `.status-dot`, `.muted` |
| Icon library | none (text + colored dot/badge vocabulary only; keep it that way) |
| Font | `--font-sans` (ui-sans-serif system stack) for labels/copy; `--font-mono` (ui-monospace stack) + `tabular-nums` for values |

Theme: dark-by-default, single theme. New panel markup goes in the static HTML shell (the no-build pattern), values filled by the existing `/api/status` poll loop — **no new fetch, no new endpoint** (D-03: dashboard folds the same `status.Report`).

| Token | Value | Usage |
|-------|-------|-------|
| xs (`--space-xs`) | 4px | Badge padding, stacked id/sub-label gaps |
| sm (`--space-sm`) | 8px | Row vertical padding, gaps between rows |
| md (`--space-md`) | 16px | Row horizontal gaps, heading bottom margin |
| lg (`--space-lg`) | 24px | Panel padding |
| xl (`--space-xl`) | 32px | Gap between top-level panels |
| 2xl (`--space-2xl`) | 48px | Content top/bottom padding |
| 3xl (`--space-3xl`) | 64px | (declared, unused this phase) |

Exceptions: 44px min-height hit targets on interactive controls (`.btn`, `.chat-link`) — existing exception, no new interactive controls this phase. 10px status dot / mem-bar height — existing fixed ornaments, not spacing.

## Typography

Existing contract — exactly 4 sizes, 2 weights. The Memory panel introduces no new sizes or weights.

| Role | Size | Weight | Line Height | Phase-23 use |
|------|------|--------|-------------|--------------|
| Body | 16px | 400 | 1.5 | Muted empty/unknown copy in the Memory panel |
| Label | 14px | 400 (600 for `.health-service` row labels) | 1.5 | Badge text, `.metric-label`, mono values |
| Heading | 20px | 600 | 1.2 | `.panel-heading` — "Memory" |
| Display | 28px | 600 | 1.2 | Header title (unchanged) |

Value formatting rules (existing, binding):
- All data values render in `--font-mono` with `font-variant-numeric: tabular-nums` (`.metric-value` / `.health-detail`) so the ~2.5 s poll never causes horizontal jitter.
- Counts use thousands grouping via the existing `groupThousands()` (e.g. `1,284,907`) — counts only, never a fabricated 0.
- Timestamps render **verbatim** (RFC3339 as stored in `recall-state.json`) in mono — no fabricated relative times ("3 minutes ago" implies a freshness computation status does not perform).

## Color

Existing 60/30/10 contract — unchanged. The Memory panel and the two new health rows consume **zero accent**.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0F1115` (`--bg-dominant`) | Page background |
| Secondary (30%) | `#1A1D24` (`--bg-secondary`) | Panels, header, dialog |
| Accent (10%) | `#3B82F6` (`--accent`) | See reserved list below — NOT used by anything Phase 23 adds |
| Destructive | `#EF4444` (`--destructive`) | Restart warning + connection banner ONLY — NOT used by skew/staleness copy |

Semantic status colors (outside the 60/30/10 budget — the dot/badge vocabulary):

| Status | Value | Phase-23 meaning |
|--------|-------|------------------|
| ready | `#22C55E` (`--status-ready`) | villa-qdrant/villa-embed health `ready`; recall index complete |
| warn | `#F59E0B` (`--status-warn`) | health `loading`; recall index incomplete; embedding-skew indicator (if surfaced) |
| down | `#EF4444` (`--status-down`) | health `down` — a REAL per-service probe result only (this phase fixes the chat-endpoint false-green; never render `ready` for a stopped service) |
| unknown | `#6B7280` (`--status-unknown`) | typed-Unknown — probe unevaluable, recall state absent/unreadable. Gray, never green, never red |

Accent reserved for (unchanged, explicit list): the **Open Chat link**, the **Switch model primary button**, **keyboard focus rings** (`:focus-visible`), and the **loaded-model row left-border**. Nothing else — the Memory panel, memory health rows, staleness badges, and skew indicator must not use `--accent`.

Honesty mapping (binding, mirrors the ROCm-readiness precedent in `readinessClass`): typed-Unknown → gray `badge-unknown`, never amber, never a fabricated value; a non-failure degraded state (incomplete index, skew) → amber `badge-warn`, NOT red — red stays reserved for genuine failure (`down`) and the destructive pair.

## Component Contract (Phase 23 additions)

### 1. Memory health rows (Health panel) — zero new UI

`villa-qdrant` and `villa-embed` arrive as ordinary entries in `report.services`; `renderHealth` already renders them XSS-safely (dot + service name + health badge + `active:` detail). **No JS, CSS, or HTML change.** The contract here is data honesty (backend): each row's health must be its OWN probe result (`ready`/`loading`/`down`/`unknown` → `unavailable` badge), per the existing `healthClass`/`healthLabel` vocabulary.

### 2. Memory panel (new) — the only new markup

- **Placement:** new `<section class="panel" id="memory-panel" aria-labelledby="memory-heading" hidden>` appended **after the Models panel** (last in `main.content`). Heading: `<h2 class="panel-heading" id="memory-heading">Memory</h2>`.
- **Visibility:** the panel ships `hidden` in the static shell (the `connection-banner[hidden]` precedent). JS removes `hidden` only when `report.memory` is present; re-hides if it disappears. Memory-off dashboards render **pixel-identical to v1.2** — the opt-in discipline made visual.
- **Body:** existing `.metric-row` rows (label muted sans left, value mono tabular right), built with `document.createElement` + `textContent` only (XSS-safe idiom — never `innerHTML` with server values):

| Row label | Value | Format |
|-----------|-------|--------|
| `embedding model` | `report.memory` model id (e.g. `nomic-embed-text-v1.5`) | mono, verbatim |
| `dimension` | e.g. `768` | mono |
| `indexed chats` | count | mono, `groupThousands()` |
| `last indexed` | completion timestamp | mono, verbatim RFC3339 |
| `recall index` | state badge (no `.metric-value`) | `.badge` per the state table below |

- **Recall-index state badge** (exact JSON field names are planner's call per CONTEXT; the visual/copy mapping is fixed):

| Data state | Badge | Class | Extra copy |
|-----------|-------|-------|------------|
| Complete run recorded | `indexed` | `badge-ready` | — |
| Run recorded but incomplete (`completed` empty or `< started`) | `incomplete` | `badge-warn` | muted line: `Last index run did not complete — re-run villa recall index.` |
| No index yet (state present, zero chats / empty timestamps) | — (no badge) | — | empty-state copy (see Copywriting Contract); count/last-indexed rows omitted, not zero-filled |
| State absent/unreadable (typed-Unknown) | `unavailable` | `badge-unknown` | muted line: `Recall index state unavailable.` Count/last-indexed rows omitted |

- **Embedding-skew indicator** (only if the planner surfaces the D-10 skew field in `report.memory` — recommended): a `.metric-row` labeled `embedding config`, value badge `mismatch` (`badge-warn`, amber NOT red) plus muted remediation line (copy below). When no confident mismatch: omit the row entirely (the honest empty state is no row — the Phase-10 image-row precedent), never a green "ok" badge for an unevaluated comparison.

### 3. Degradation behavior (existing conventions, binding)

- Status poll failure → global red connection banner + `.stale` dimming of last-good values (D-11 pattern); the Memory panel keeps last-good content dimmed, it does not blank.
- If memory-row health probes are TTL-cached backend-side (RESEARCH Pitfall 2 mitigation), the UI needs **no spinner and no cadence indicator** — last-good + stale-dim already covers bounded staleness honestly.
- `prefers-reduced-motion` rule already global; no animation may be added.

### 4. Accessibility

- Panel follows the existing `aria-labelledby` pattern; badges carry text (never color-only meaning); no new interactive controls → no new focus/hit-target work. The existing `aria-live="polite"` stays confined to the header verdict.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | **None new this phase.** The existing `Switch model` dialog/CTA is unchanged (D-09: chat swap leaves the memory stack intact — no dialog copy change needed) |
| Memory panel heading | `Memory` |
| Empty state heading | `No recall index yet` (rendered as `.model-empty-heading`-weight 600 line, mirroring the Models empty state) |
| Empty state body | `Build it with villa recall index — indexed chats appear here once a run completes.` (muted) |
| Unknown state (typed-Unknown) | badge `unavailable` + muted `Recall index state unavailable.` — never a fabricated count or a red badge |
| Incomplete-index state | badge `incomplete` + muted `Last index run did not complete — re-run villa recall index.` |
| Skew indicator (if surfaced) | badge `mismatch` + muted `Configured embedding model differs from the indexed vectors — re-index with villa recall index --rebuild, or revert config.toml.` |
| Service health rows | existing vocabulary verbatim: `ready` / `loading` / `down` / `unavailable` + `active: <state>` |
| Error state (poll failure) | existing global banner copy unchanged: `Lost connection to the dashboard. Retrying…` |
| Destructive confirmation | **None new in the dashboard.** The CLI-side destructive/confirm copy (restore dimension-skew WARN+confirm, `recall index` refusal) follows the refuse-with-remediation strings in 23-RESEARCH.md §Code Examples — out of this UI contract's surface, in the planner's |

Copy rules (existing, binding): sentence case; backtick-free in DOM text (the dashboard renders plain text); every degraded state names the next step; never claim freshness or success the data cannot prove.
