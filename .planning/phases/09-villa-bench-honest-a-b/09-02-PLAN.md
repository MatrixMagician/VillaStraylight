<objective>
Build `internal/bench` — the pure, Deps-injected benchmarking core, cloned from the `internal/backendswap` idiom (Deps/Result/Run). It runs warmup-discard → N residency-checked measured runs → separate pp/tg median+stddev over kept runs only; voids non-resident runs; surfaces a void-exhaustion WARN when too few resident runs are collected; and, for `--ab`, applies an identical BenchSpec to both sides and ALWAYS restores the original backend via injected `Switch`/`Restore` (which the live layer delegates to `backendswap.Run` — never re-implemented here). Print-free and exit-free: the cobra layer (Plan 03) owns presentation.

Purpose: Honest methodology (BENCH-02) and separated pp/tg (BENCH-01) implemented as pure, golden-testable arithmetic with no live host — the only genuinely new code in this composition phase.
Output: `internal/bench` package + a fake-Deps recorder test suite covering every methodology invariant.
</objective>
