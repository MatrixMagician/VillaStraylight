## Trust Boundaries

| Boundary | Description |

| `<backend>` --ab target → inference.BackendFor | untrusted CLI arg crosses into the fail-closed backend allowlist (same as `backend set`) |
| --ab Switch/Restore → backendswap.Run | the ONLY mutation path; transactional rollback owned by the Phase-8 (already threat-reviewed) core |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Spoofing / Validation | `--ab` backend target | mitigate | the target crosses only into `inference.BackendFor` (fail-closed allowlist) — an unknown target is a prove/resolve fail, never a silent fallback (clone liveProve:68). `-n`/`--n-predict` are bounded ints. |
| T-09-07 | Information disclosure | bench output | mitigate | the render + --json carry ONLY numeric timings + Kept/Void + stated-conditions — never the prompt or sampling params (mirror metrics.Slot field-set discipline). |
| T-09-08 | Denial of service | wedged/CPU-fallback server hanging a run forever | mitigate | every run bounded by `context.WithTimeout(ctx, spec.Timeout)` (mirror proveTimeout=5m, the load_tensors-hang guard); a timed-out run is void, never an infinite wait. |
| T-09-09 | Tampering / Availability | `--ab` leaving the stack on a non-default backend | mitigate | Switch/Restore delegate to `backendswap.Run` (transactional rollback); the core's always-restore (09-02) plus TestBenchABRestoresOriginal guarantee the original backend is restored; loopback-only posture unchanged by a flip. |
| T-09-10 | (project invariant) Tampering | backend-marker literal leak in cmd/villa/bench.go | mitigate | markers arrive ONLY via `inference.BackendFor(target).ResidencyProof()`; the cmd/villa-walking TestSeamGrepGate gates every non-test .go (seam_test.go:127-141) — bench.go stays literal-free. |
