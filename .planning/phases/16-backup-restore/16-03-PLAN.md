<objective>
Ship `villa restore <archive>`: a transactional restore that mirrors `backendswap.Run` — read+verify the archive (fail-closed BLOCK on corruption/incompatible schema), skew-check with WARN-and-confirm (BAK-03), capture the current state strictly before mutating, quiesce Open WebUI, apply (restore config via `config.SaveVilla`, restore data-dir artifacts atomically, recreate the OWUI volume CLEAN then `podman volume import`), restart, prove offload-honestly, and roll back verbatim on any failure with honest rollback-complete/incomplete reporting.

Purpose: deliver BAK-02 (transactional restore that never corrupts a running stack) and BAK-03 (skew warning before apply). The single biggest plan-shaping fact — `podman volume import` MERGES and does NOT auto-create — is honored by the mandatory clean-recreate (rm not-found-tolerant → Quadlet recreate → ensure volume exists) BEFORE import, on BOTH the forward apply AND the rollback path.
Output: `internal/backup/restore.go` Restore() (pure clone of the backendswap frame), `cmd/villa/restore.go` (live wiring + y/N consent + `--yes`/`--force`), tests, command registered in root.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| archive bytes → filesystem/podman | The restore archive is UNTRUSTED input: every entry path + checksum + manifest schema must be validated before any write |
| restore mutation → running stack | A partial/failed restore must never corrupt the running stack (transactional capture+rollback) |
| volume/service names → podman/systemd exec | Must be fixed-arg, config/catalog-resolved — no shell interpolation |
| restore prove → success claim | Health-200/is-active alone must never be reported as success (offload-asserting) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-03a | Tampering / EoP | tar extraction | mitigate | tar-slip guard (readArchive/assertInsideDir) refuses any entry escaping the extraction dir — tested with "../" + absolute entries |
| T-16-03b | Tampering | corrupt/tampered archive | mitigate | per-entry SHA-256 verify before apply; fail-closed BLOCK on mismatch (zero side effects) |
| T-16-03c | Tampering | incompatible/future manifest schema | mitigate | fail-closed BLOCK on unreadable/newer manifest.schema_version (mirror usage.Load future-fail-closed) |
| T-16-03d | Tampering / EoP | destructive restore corrupting a running stack | mitigate | transactional capture-before-mutate + verbatim rollback (BAK-02); a partial restore leaves the stack intact, rollback-incomplete reported honestly |
| T-16-03e | Tampering (integrity) | stale state leak via merge-import | mitigate | clean-recreate the OWUI volume (VolumeRm + Quadlet recreate + EnsureVolume) BEFORE VolumeImport, on apply AND rollback |

| T-16-03g | Repudiation (false success) | restore prove | mitigate | offload-asserting prove = FAIL → rollback; health-200/is-active never reported as success (D-07) |
