<objective>
Build the pure `internal/backup` core and the two additive seam gaps (the exported OWUI image accessor + a build-stamped villa version constant) that the `villa backup` / `villa restore` commands compose in Waves 2–3.

This plan delivers all host-I/O-free logic: the `Manifest` schema-versioned doc, SHA-256 compute/verify, outer-tar assembly + tar-slip-guarded extraction, the injectable `Deps`/`Result` types, and the pure skew comparison. It is the foundation — no podman, no filesystem mutation, fully off-hardware testable via `fakeDeps`.

Purpose: lock the contracts (manifest fields, Deps func-field shape, Result type, prove sentinel) so the cmd-tier plans wire seams against a fixed interface rather than discovering it. Mirrors the pure-core + injectable-seam pattern proven in `backendswap`/`usage`/`status`.
Output: `internal/backup/{backup,manifest,checksum,tarutil,deps}.go` + tests; `orchestrate.OpenWebUIImage()`; `cmd/villa/version.go` + Makefile `-ldflags` stamp.

Note on store schema versions: the manifest carries `UsageSchemaVersion`/`BenchSchemaVersion` as plain `int` fields populated from `ManifestInput` (pure assembly only, no I/O). The REAL values are sourced by the cmd-tier from NEW exported accessors `usage.SchemaVersion()` / `benchstore.SavedReportSchemaVersion()` — added in Plan 16-02 Task 1, because the underlying consts (`usageSchemaVersion`, `savedReportSchemaVersion`) are unexported and unreachable from this pure core. This plan defines the int fields; Plan 02 supplies the values.
</objective>
