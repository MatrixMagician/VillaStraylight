<objective>
Ship `villa backup`: stop Open WebUI for a clean SQLite copy, `podman volume export` the OWUI data volume, build the seam-sourced manifest (versions, both image digests, store schema versions, per-entry SHA-256, excluded model identities), assemble the single plain `.tar` (manifest.json + config.toml + openwebui-volume.tar + usage.json + the single bench-reports.jsonl), restart OWUI, and write the archive 0600 with a traversal-guarded output path.

This plan also closes the store-schema reachability gap: it adds two exported one-line accessors — `usage.SchemaVersion()` and `benchstore.SavedReportSchemaVersion()` — over the currently-unexported `usageSchemaVersion` / `savedReportSchemaVersion` consts, so the cmd-tier can supply the REAL store schema versions into `manifest.UsageSchemaVersion`/`BenchSchemaVersion` (Plan 01 defined those as plain int fields).

Purpose: deliver BAK-01 end-to-end — a single self-describing local archive that excludes re-pullable model weights and records their identity for re-pull. The pure `Backup()` orchestrator lives in `internal/backup` over the Task-1 `Deps`; all podman/systemd/file I/O is injected via `liveBackupDeps` in `cmd/villa`.
Output: `cmd/villa/backup.go` (+ shared `cmd/villa/podman_volume.go`), `internal/backup/backup.go` Backup(), the two store accessors, tests, and the command registered in root.

Deferred within scope (D-13): `villa backup`/`villa restore` ship human-readable output only this phase; the optional `--json` is discretionary per CONTEXT.md D-13 and is intentionally NOT implemented here. If added later it is a NEW, separate frozen golden contract (its own schema_version under `cmd/villa/testdata/`), NEVER an evolution of `status.Report`.
</objective>
