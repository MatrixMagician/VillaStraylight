<objective>
Add the BENCH-03 persistence write-hook to the cmd-tier `villa bench` verb: after a successful bench run is measured and printed, fold the pure `bench.Result` (pp/tg, VoidExhausted/Reason) plus a cmd-tier-captured comparability `Fingerprint` (model/quant/ctx from config, host gfx id from `detect.Probe()` `.Known`-guarded) into a `benchstore.SavedReport` and append it as one JSONL line via a new `liveBenchstoreDeps` seam (O_APPEND|O_CREATE|O_WRONLY 0600 under a 0700 dir, traversal-guarded). The write is LOUD-but-NON-FATAL on failure — a persistence error never changes the measurement's exit code.

Purpose: BENCH-03 — persist each `villa bench` run as a versioned saved report under XDG, pp/tg separate, residency-void state recorded.
Output: extended `cmd/villa/bench.go` + `cmd/villa/bench_test.go`.
</objective>
