<objective>
Build `internal/backendswap` — the pure, `Deps`-injected transactional core for `villa backend set`: capture→mutate→prove→rollback. This is the heart of Phase 8 and the single genuinely new mechanism in the milestone (the transactional rollback frame wrapped around the proven `modelswap` forward skeleton).

Purpose: A backend switch on a running install must NEVER leave a broken or silently-CPU stack. The core captures the verbatim prior unit+config before mutating, gates cutover on an injected Prove verdict (real generation-probe + residency proof), and auto-rolls back verbatim on any failure (BSET-02), while preserving the configured model and refusing-with-remediation when it no longer fits or ROCm preflight blocks (BSET-01).

Output: `internal/backendswap/backendswap.go` (Deps, Result, ProveVerdict, Run) + `internal/backendswap/backendswap_test.go` (Wave 0 state-machine suite). Pure package, no live host, literal-free of backend tokens.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arg → core | The `<backend>` target string crosses into `Run`; it is validated by the allowlist resolver in the live Deps (Plan 02), never interpolated into a shell/path here. |
| core → filesystem (Quadlet dir) | Capture/restore read+write `villa-llama.container` bytes; writes go through the injected `RestoreUnit` seam (live impl uses the traversal-guarded `orchestrate.WriteUnits`). |
| core → systemd (user manager) | daemon-reload/restart cross into the rootless user systemd via injected fixed-arg seams (live `orchestrate.Systemd`, no shell). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8-01 | Tampering | capture-before-mutate ordering in `Run` | mitigate | Capture is step 4, strictly before any SaveConfig/ReconcileAndWrite/Restart; `TestCaptureBeforeMutate` asserts `capture` index < `save`/`write` index (Pitfall 4). An uncapturable prior unit refuses without mutating. |
| T-8-02 | Denial of Service | failed/degraded switch leaving a broken stack | mitigate | Verbatim rollback restores the exact captured prior unit+config and re-readies; `TestRollbackVerbatim`/`TestMutateErrorRollsBack` assert byte-equal restore (SC#2 — a failed switch is a no-op to the running stack). |
| T-8-03 | Spoofing (of health) | cutover gate trusting is-active/200 | mitigate | `Run` switches ONLY on `Prove.Status == ProveStatusPass`; any non-pass verdict rolls back (`TestActiveNotSuccess`). The residency/generation content of the verdict is supplied by Plan 02's `liveProve` (SC#3). |
| T-8-04 | Tampering | backend marker literals leaking into the core | mitigate | backendswap is literal-free of `ROCm0`/HSA/fault/image tokens; markers arrive only through the injected `Prove` seam. `TestSeamGrepGate` (inference/seam_test.go) walks `internal/` and fails on any leak; the acceptance grep double-checks. |
| T-8-05 | Tampering | rollback partial-failure presented as clean | accept→report | Rollback is best-effort with accumulated errors; an incomplete rollback is reported honestly (`TestRollbackIncompleteReported`, Pitfall 5) rather than masked. No silent success on a half-restored stack. |
</threat_model>
