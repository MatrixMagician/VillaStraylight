<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI arg → resolver | The `<backend>` target crosses into `liveBackendSwapDeps`/`liveProve`; it is validated by `inference.BackendFor` (fail-closed allowlist: vulkan/rocm/empty), never interpolated into a shell or path. |
| liveProve → running server | A real generation probe + journald residency read cross into the restarted `villa-llama.service`; the verdict content (markers) is sourced only from `BackendFor(target).ResidencyProof()`. |
| live Deps → filesystem / systemd | Capture/restore + reload/restart go through the traversal-guarded `orchestrate.WriteUnits` and fixed-arg `orchestrate.Systemd` seams (no shell). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8-06 | Spoofing (of health) | a degraded ROCm backend faking a successful cutover | mitigate | `liveProve` requires pollHealth ready AND a real generation token AND `RunningOffloadVerdict == StatusPass` (ROCm0 buffer line + non-zero gpu_busy DURING the decode + no Memory access fault); a CPU-fallback / 0%-busy / fault run is StatusFail → rollback (SC#3; Pitfalls 1/3). |
| T-8-07 | Denial of Service | unbounded `load_tensors` hang wedging the CLI | mitigate | The readiness poll is bounded by `proveTimeout` (context.WithTimeout, seeded from defaultReadyTimeout); deadline expiry is a PROVE FAIL → rollback, never an infinite wait (Pitfall 2). |
| T-8-08 | Tampering | command injection via the `<backend>` arg | mitigate | `inference.BackendFor` allowlist resolves the target; fixed-arg `systemctl`/`journalctl` via `orchestrate.Systemd` (no shell). |
| T-8-09 | Tampering | path traversal on unit capture/restore | mitigate | CaptureUnit reads inside `quadletUnitDir()`; RestoreUnit writes via `orchestrate.WriteUnits` which re-guards with `assertInsideDir` (reconcile.go). |
| T-8-10 | Information disclosure | loopback→exposed port regression on re-render | accept | The renderer/PublishPort privacy posture is unchanged by this phase; the byte-frozen status golden + loopback assertion guard it (out of this phase's mutation surface). |
| T-8-11 | Tampering | ROCm0/HSA/fault literals leaking into cmd/villa | mitigate | `backend.go` is literal-free; markers arrive only via `BackendFor(target).ResidencyProof()`; acceptance grep enforces. |
</threat_model>
