<objective>
Create the new pure `internal/usage` core: a reset-aware `Fold(prior, sample) -> Totals`
over monotonic `_total` counters (NOT rate gauges), keyed per model, with an injected
byte-I/O store seam plus a self-contained XDG data-dir resolver, path-traversal guard, and
full-file atomic temp+rename persistence. No host I/O in the fold; persistence is a seam.

Purpose: This is the accumulation engine for USAGE-01. Every downstream plan (metrics feed,
status surface, dashboard writer) composes this core. Per D-01 it is pure arithmetic; per
D-02 the store lives under XDG _data_ (not config, not cache) with the proven 0600/0700 +
traversal-guard + atomic-rewrite discipline.

Output: `internal/usage/usage.go` (types + Fold + store seam + resolver + guard + atomic
write) and `internal/usage/usage_test.go` (fold/round-trip/path/security tests).

| T-15-02 | Tampering / DoS | usage.json under concurrent rewrites | mitigate | full-file atomic temp+rename (os.CreateTemp same dir → os.Rename) — a crash mid-write never leaves a torn file; the sole-writer mutex is added in Plan 04 |
