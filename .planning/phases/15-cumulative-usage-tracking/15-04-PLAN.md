usage store (Plan 01) and atomically writes — capturing `cfg.Model` inside the critical

cumulative totals read-only in the dashboard UI from the SAME `status.Report.usage` field
(Plan 03) — no new endpoint, inside the existing Performance panel.

Purpose: This closes USAGE-02's dashboard surface + the D-07 sole-writer contract + the
approved UI-SPEC. The live tok/s view and the live `/api/metrics` payload stay unchanged
(D-10: cumulative rides on the existing `/api/status` poll, additive).

Output: dashboard server.go (+usageMu/+seams), api.go (handleMetrics fold+write), api_test.go,
cmd/villa/dashboard.go (live wiring), dashboard.js (cumulative rows).
</objective>
