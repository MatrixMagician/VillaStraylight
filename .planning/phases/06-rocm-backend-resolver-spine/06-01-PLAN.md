<objective>
Build the backend-neutral residency-proof engine: extend the `Backend` interface
with `ResidencyProof() ResidencyMarkers` (D-04), implement Vulkan's markers
byte-identically (D-05), and parameterize BOTH Vulkan-hardcoded scrape paths
(`offload.go` start-time + `running_offload.go` running) by the descriptor so a
ROCm backend can slot in (Plan 02) without re-rolling the offload math.

Purpose: This is the residency spine. Until both scrapers key off `ResidencyMarkers`
instead of `"Vulkan0"`/`"ggml_vulkan:"`/`"- Vulkan"`, a ROCm offload-assert would
silently stay Vulkan-only on the install/`villa inference` path. Vulkan behavior
must not change ā€” the existing Vulkan tests are the regression guard (SC#4).

Output: `ResidencyMarkers` struct + `ResidencyProof()` on `Backend` + Vulkan impl;
`scrapeOffloadLog(stderr, m)` and `scrapeLoadTensorsResidency(journal, m)`;
`ValidateInput`/`RunningOffloadInput` threaded with markers; a fault-scan rule and
a start-time N<M partial-FAIL rule; existing Vulkan tests unchanged and green.
</objective>
