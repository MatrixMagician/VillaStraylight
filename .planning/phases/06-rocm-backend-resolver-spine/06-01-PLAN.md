<objective>
Build the backend-neutral residency-proof engine: extend the `Backend` interface
with `ResidencyProof() ResidencyMarkers` (D-04), implement Vulkan's markers
byte-identically (D-05), parameterize BOTH Vulkan-hardcoded scrape paths
(`offload.go` start-time + `running_offload.go` running) by the descriptor so a
ROCm backend can slot in (Plan 02) without re-rolling the offload math, and wire
the `gpu_busy_percent` residency signal (D-06) into the running verdict as a
corroborating signal folded through `combineOffload`.

Purpose: This is the residency spine. Until both scrapers key off `ResidencyMarkers`
instead of `"Vulkan0"`/`"ggml_vulkan:"`/`"- Vulkan"`, a ROCm offload-assert would
silently stay Vulkan-only on the install/`villa inference` path. And per ROADMAP
SC#2 + ROCM-02 + D-06 the residency proof is a 4-signal assert — `ROCm0` buffer
line + `offloaded N/N` + non-zero sysfs `gpu_busy_percent` during a decode +
absence of `Memory access fault by GPU node`. This plan wires the busy signal as a
verdict INPUT now (off-hardware, fixture-driven, WARN-capable per D-07); the live
decode-time read is the Phase-8 follow-on. Vulkan behavior must not change — the
existing Vulkan tests are the regression guard (SC#4); Vulkan supplies no busy
reading so the field defaults to Unknown, the busy fold is skipped, and existing
Vulkan verdicts are unchanged.

Output: `ResidencyMarkers` struct + `ResidencyProof()` on `Backend` + Vulkan impl;
`scrapeOffloadLog(stderr, m)` and `scrapeLoadTensorsResidency(journal, m)`;
`ValidateInput` threaded with markers; `RunningOffloadInput.GPUBusyPercent detect.Int`
+ a `gpuBusyFloor` rule folded via `combineOffload`; a fault-scan rule and a
start-time N<M partial-FAIL rule; existing Vulkan tests unchanged and green.
</objective>
