<objective>
Fix the `villa bench` single-mode backend label. In single-backend mode the measured
backend is never named: the human header prints `backend ():` and `--json`
`single.backend` is `""`. Thread the configured backend into the single-mode result so
both the rendered header and the `--json` `single.backend` field name the measured
backend.

Purpose: honesty/completeness — a single-mode bench band must state which backend it
measured (UAT Test 1 gap, 09-UAT.md). Cosmetic to the numbers; does NOT touch the
pp/tg-separate contract.

Output: a one-line wiring fix in the cmd layer (`runBench`), an extended cmd test
asserting the label is present in both human and JSON output, and an unchanged --ab
golden.
</objective>

  Leave the --ab branch untouched.

  (`liveBenchDeps` Measure closure uses `liveMeasure(ctx, cfg.Backend, spec)`,
  cmd/villa/bench.go:215). But `runBench` is driven directly in tests with a stubbed

  unconditionally (that would read the developer's real config / fail in CI).

  (`benchEndpointReachable`, cmd/villa/bench.go:453) overridden in tests via
